# Product Launch Checklist

Shipping the app is only part of launching the product.

A product is not launch-ready just because the core feature works. It also needs the surrounding product layer that helps users understand it, trust it, try it, get support, and know what happens next.

## 1. Product Clarity

Before launch, a user should be able to answer:

- What does this product do?
- Who is it for?
- What problem does it solve?
- Why does it matter now?
- What do I get after signing up?
- What happens next?

If the user needs a call, demo or explanation from the founder to understand the product, the product page is not clear enough.

## 2. Trust and Legal Basics

Every product should have:

- Privacy policy
- Terms of service
- Cookie policy where needed
- Contact route
- Data deletion process
- Clear company ownership
- Support email
- Refund/cancellation explanation where relevant

For products using AI, generated legal or compliance material should be clearly described as draft output unless professionally reviewed.

## 3. Onboarding

A launch-ready product needs a first-use path.

Check:

- Can a user sign up?
- Can a user understand the first action?
- Is there an empty state?
- Is there a sample/demo path?
- Is the user told what to do next?
- Are errors written in plain language?
- Can the user recover if something goes wrong?

## 4. Support Readiness

Before launch, make sure there is:

- Support inbox
- FAQ
- Known issues list
- Basic help page
- Bug reporting route
- Internal owner for support
- Response expectation

A user should never feel abandoned after trying the product.

## 5. Commercial Readiness

Check:

- Pricing page exists
- Free/paid boundary is clear
- Subscription/cancellation flow works
- Payment flow is tested
- Confirmation emails work
- Account deletion works
- Receipts/invoices are handled where needed

If users can pay, they must also understand how to cancel, get help and manage their account.

## 6. Growth Readiness

Before launch, prepare:

- Landing page
- Product screenshots
- Founder announcement post
- Product announcement post
- Short demo clip
- Social profile links
- Search Console
- Analytics
- Basic conversion tracking
- Email capture where useful

A launch without a distribution path is just a deployment.

## 7. Operational Readiness

Check:

- Monitoring is enabled
- Errors are visible
- Backups are understood
- Deployment rollback is possible
- Owner knows how to respond to incidents
- Logs do not expose secrets
- Sensitive data handling is clear

## 8. Security and Privacy Sanity Check

Before launch, confirm:

- No secrets are exposed in public repos
- No API keys are visible in frontend code
- Authentication works as expected
- Password reset works if applicable
- HTTPS is enforced
- Basic security headers are set
- Admin routes are protected
- User data is not logged unnecessarily

## 9. Final Launch Question

Can a real user arrive cold, understand the product, trust it, start using it, get help, and know what happens next?

If not, the product is not launch-ready yet.

## Principle

The product is not just the code.

The product is the full experience around the code:

- trust
- onboarding
- support
- pricing
- legal
- communication
- measurement
- operations

That is the layer many teams underestimate.
