# Contributing

This repository contains public Conso4s product playbooks, templates and principles.

Contributions are welcome when they improve clarity, usefulness or practical application.

## Useful contributions

Good contributions include:

- clearer wording
- practical checklist improvements
- additional examples
- corrections
- better structure
- useful templates
- missing operational considerations

## Not suitable for this repository

Please do not submit:

- production source code
- customer data
- credentials or secrets
- private security findings
- exploit code
- vendor spam
- promotional content
- legal advice presented as final guidance
- sensitive internal architecture details

## Contribution principle

The material in this repository should stay practical, factual and useful.

The goal is not to create theory or marketing copy.

The goal is to help teams build, launch, operate and improve products that work in the real world.

## Before opening a pull request

Please check that your change:

- is clear
- is practical
- does not expose sensitive information
- does not make exaggerated claims
- does not create legal, security or compliance confusion
- fits the purpose of the repository

## Reporting issues

If you spot an error, unclear wording or a missing checklist item, open an issue with:

- the file name
- the section
- what is unclear or wrong
- the suggested improvement

## Principle

Ideas are only real when they work in production.

Everything else is noise.
