# Security and Privacy Stance

This repository contains public product principles, examples and playbooks.

It does not contain production source code, customer data, secrets, credentials, API keys, private infrastructure details or internal product logic.

## Public Repository Rules

We do not publish:

- customer data
- private source code
- production secrets
- API keys
- access tokens
- infrastructure credentials
- private vulnerability details
- exploit code
- sensitive security findings

## Product Principle

Security and privacy claims should be factual, specific and testable.

We avoid exaggerated claims such as guaranteed security, absolute protection or certification-style language unless it is formally true and evidenced.

## Data Principle

Products should minimise sensitive data collection, make user expectations clear, and avoid storing information that is not needed for the job.

Where sensitive information may appear in a workflow, the product should make the safe behaviour obvious.

## AI and Data Handling

AI products should not treat user data casually.

Where AI is used, the product should make clear:

- what is being processed
- why it is being processed
- what the user receives
- where human review is needed
- what should not be submitted

Users should not submit passwords, API keys, private credentials, production secrets or sensitive personal data unless the product is explicitly designed and documented for that purpose.

## Trust

Trust is not created by slogans.

It is created through clear language, safe defaults, honest limits, predictable behaviour and respectful data handling.
